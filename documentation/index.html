Design Approach — How jqPlot Uses JavaScript Objects and Arrays

jqPlot is configured entirely through plain JavaScript objects and arrays passed to a single factory function. There is no DOM manipulation required on your part — you describe what you want in a configuration object, and jqPlot figures out how to draw it. Understanding this structure unlocks the full library.

The Factory Call

Every chart starts with one function call:

$.jqplot( targetId, seriesData, options );
ArgumentTypeRole
targetIdStringID of the container <div>
seriesDataArray of ArraysThe data — one inner array per series
optionsObjectEverything else: axes, series styling, plugins, legend…

Series Data — Array of Arrays

The second argument is always an array of arrays. Each inner array is one series. A series is itself an array of data points. A data point is either a plain number (jqPlot infers the x from its index) or a two-element array [x, y]. 

// Three series, each with implicit x = 1,2,3,4
$.jqplot('chart', [
  [10, 20, 15, 30],          // series 1
  [25, 18, 22, 28],          // series 2
  [5,  12,  8, 18]           // series 3
]);

// Explicit [x, y] pairs — required for date axes or sparse data
$.jqplot('chart', [
  [['2024-01-01', 12.4], ['2024-06-01', 18.9], ['2025-01-01', 24.7]]
]);
The outer array is always present even for a single series: $.jqplot('id', [ [1,2,3] ]) — not $.jqplot('id', [1,2,3]).

The Options Object — Nested Structure

The third argument is a plain object whose top-level keys map to chart subsystems. Each subsystem is itself an object with its own keys. This nesting is consistent throughout the library. 

$.jqplot('chart', [data], {

  // ── Top-level keys ────────────────────────────────────────────
  title:  'My Chart',         // string or object
  legend: { show: true, location: 'nw' },
  grid:   { shadow: false, background: '#fafbfc' },

  // ── seriesDefaults: applied to ALL series ─────────────────────
  seriesDefaults: {
    renderer:        $.jqplot.BarRenderer,   // swap the draw engine
    rendererOptions: { barPadding: 4 },      // options for that renderer
    showMarker:      false
  },

  // ── series: array, one object per series (overrides defaults) ─
  series: [
    { label: 'Revenue', color: '#2980b9', lineWidth: 2.5 },
    { label: 'Cost',    color: '#e74c3c', lineWidth: 1.5 }
  ],

  // ── axes: object keyed by axis name ───────────────────────────
  axes: {
    xaxis: {
      renderer:    $.jqplot.DateAxisRenderer,
      tickInterval:'1 year',
      tickOptions: { formatString: '%Y' }
    },
    yaxis: {
      label:         'Revenue ($M)',
      labelRenderer: $.jqplot.CanvasAxisLabelRenderer,
      min:  0,
      tickOptions: { formatString: '$%.1fM' }
    }
  }
});

Renderers — Swapping the Drawing Engine

jqPlot's most powerful pattern is the renderer system. Almost every visual component — series, axes, ticks, legend — has a default renderer that can be replaced by assigning a different class to the renderer property. The renderer is a constructor function; jqPlot instantiates it internally.

Series renderers

Set on seriesDefaults.renderer or per-series in the series array.

seriesDefaults: {
  renderer: $.jqplot.BarRenderer
}
// or per-series:
series: [
  { renderer: $.jqplot.OHLCRenderer,
    rendererOptions: { candleStick: true } }
]

Axis renderers

Set on the axis object inside axes.

axes: {
  xaxis: {
    renderer: $.jqplot.DateAxisRenderer
  },
  yaxis: {
    renderer: $.jqplot.LogAxisRenderer
  }
}

Plugins — Self-Registering via Hooks

Interactive plugins (Highlighter, Cursor, Draggable, CanvasOverlay, dateEnhancement) are loaded via <script> tags and self-register by pushing handler functions onto global hook arrays. You never call them directly — jqPlot's draw pipeline invokes them at the right moment. 

// Inside a plugin source file — this runs at script-load time:
$.jqplot.postDrawHooks.push($.jqplot.CanvasOverlay.postPlotDraw);
$.jqplot.preParseOptionsHooks.push($.jqplot.DateEnhancement.preParseOptions);

You activate a plugin by including its options object in the chart options. If the plugin key is absent, the plugin is loaded but does nothing. 

// Highlighter activates when this key is present:
$.jqplot('chart', [data], {
  highlighter: {
    show:        true,
    tooltipAxes: 'both',
    formatString:'<strong>x=%s  y=%s</strong>'
  }
});

The Hook Pipeline

jqPlot's internal lifecycle fires hook arrays at key moments. Plugins use these to inject behavior without modifying core:

1

$.jqplot.preInitHooks

Called before the plot object is initialized. Args: (target, data, options). Use to mutate options before they are parsed.

2

$.jqplot.preParseOptionsHooks

Called just before gridPadding and other layout options are merged. Args: (options) — one argument only, this is the plot object. Use to adjust layout before the grid is sized.

3

$.jqplot.postInitHooks

Called after the plot object is fully initialized. Args: (target, data, options).

4

$.jqplot.preDrawHooks

Called before canvas drawing begins. Use to modify series data or axis bounds.

5

$.jqplot.postDrawHooks

Called after all canvas drawing is complete. Args: none — this is the plot object. The primary hook for overlay plugins (CanvasOverlay, dateEnhancement). At this point plot._gridPadding, plot.axes.xaxis.series_u2p() and all layout values are finalized.

Hook signature pitfall: preParseOptionsHooks receives only (options) — not (target, data, options) like preInitHooks. Declaring the wrong number of parameters is a silent error that causes options to be misread and can crash the entire plot initialization.

Coordinate Conversion — Data Units to Pixels

After a plot is drawn, each axis exposes series_u2p(value) which converts a data-unit value to a pixel offset relative to the grid origin (plot._gridPadding.left). To get an absolute canvas pixel: 

var plot  = $.jqplot('chart', [data], options);

// After draw, inside a postDrawHooks handler:
var xPx = plot._gridPadding.left + plot.axes.xaxis.series_u2p(dataValue);
var yPx = plot._gridPadding.top  + plot.axes.yaxis.series_u2p(dataValue);

// For DateAxisRenderer axes, convert the date string to a timestamp first:
var ts  = new $.jsDate('2024-06-01').getTime();
var xPx = plot._gridPadding.left + plot.axes.xaxis.series_u2p(ts);

Key Object Reference

Object / PropertyTypeDescription
plot._gridPaddingObject{top, right, bottom, left} — pixel offsets from canvas edge to grid boundary
plot._plotDimensionsObject{width, height} — total canvas pixel dimensions
plot.axes.xaxisObjectThe x-axis instance. Has .min, .max, .series_u2p(), .renderer
plot.series[n]ObjectSeries instance. Has .data, .color, .renderer
plot.optionsObjectThe merged options object — includes plugin options like dateRanges
plot.targetjQueryThe container element — use to inject DOM labels with plot.target.append($lbl)
plot.eventCanvas._elemjQueryThe top-most canvas that receives mouse events. Bind mouse handlers here.

Quick Start

<!-- 1. Load jQuery -->
<script src="https://cdnjs.cloudflare.com/ajax/libs/jquery/3.7.1/jquery.min.js"></script>

<!-- 2. Load jqPlot core -->
<link  rel="stylesheet" href="dist/jquery.jqplot.min.css">
<script src="dist/jquery.jqplot.min.js"></script>

<!-- 3. Load only the plugins you need -->
<script src="dist/plugins/jqplot.barRenderer.min.js"></script>
<script src="dist/plugins/jqplot.categoryAxisRenderer.min.js"></script>

<!-- 4. A container div -->
<div id="my-chart" style="width:600px;height:300px;"></div>

<script>
$(document).ready(function() {
  $.jqplot('my-chart', [[48, 63, 71, 55, 82]], {
    seriesDefaults: {
      renderer: $.jqplot.BarRenderer,
      rendererOptions: { fillToZero: true }
    },
    axes: {
      xaxis: {
        renderer: $.jqplot.CategoryAxisRenderer,
        ticks: ['Jan','Feb','Mar','Apr','May']
      }
    }
  });
});
</script>
Always wrap $.jqplot() calls inside $(document).ready() — the container <div> must exist in the DOM before jqPlot can measure it.

Build Pipeline

Source files live in src/. The dist/ directory is gitignored and must be regenerated after any source change. 

npm install       # first time only — installs esbuild and clean-css
npm run build     # every time src/ changes — writes to dist/
SourceBuilt outputWhen to rebuild
src/plugins/jqplot.*.jsdist/plugins/jqplot.*.min.jsAny plugin source change
src/jquery.jqplot.jsdist/jquery.jqplot.min.jsCore source change
src/jquery.jqplot.cssdist/jquery.jqplot.min.cssCSS change
Changes to demo/*.html or docs/*.html do not require a rebuild — they are served directly. Only changes to src/ require npm run build.
