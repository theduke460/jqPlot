Pie & Donut Charts

Circular charts with slice labels, configurable start angles, and donut center totals.

Required Plugins

<script src="dist/plugins/jqplot.pieRenderer.min.js"></script>
<script src="dist/plugins/jqplot.donutRenderer.min.js"></script>

Pie Chart

Data is a single array of [label, value] pairs. Slices are sized proportionally to their values. startAngle: -90 starts the first slice at 12 o'clock. 

var slices = [
  ['Flights',  42],
  ['Hotels',   28],
  ['Car Hire', 18],
  ['Other',    12]
];

$.jqplot('pie-chart', [slices], {
  seriesDefaults: {
    renderer: $.jqplot.PieRenderer,
    rendererOptions: {
      showDataLabels:    true,
      dataLabels:        'value',    // 'value' | 'percent' | 'label'
      startAngle:        -90,
      sliceMargin:       4,
      shadow:            true,
      shadowOffset:      2
    }
  },
  legend: {
    show:             true,
    location:         'e',
    rendererOptions:  { numberColumns: 1 }
  }
});

Slice Label Styling

Slice labels are injected as DOM <span> elements with class jqplot-pie-series jqplot-data-label. Style them with CSS: 

.jqplot-pie-series.jqplot-data-label {
  font-weight: bold;
  color: #fff;
  text-shadow:  1px  1px 0 #000,
               -1px -1px 0 #000,
                1px -1px 0 #000,
               -1px  1px 0 #000;
}

Donut Chart

Same data format as pie. DonutRenderer adds a hole in the centre. innerRadius controls hole size as a percentage of the outer radius. Set totalLabel: true to display the sum in the centre. 

$.jqplot('donut-chart', [slices], {
  seriesDefaults: {
    renderer: $.jqplot.DonutRenderer,
    rendererOptions: {
      innerRadius:       55,       // % of outer radius (controls hole size)
      startAngle:        -90,
      sliceMargin:       3,
      showDataLabels:    true,
      dataLabels:        'value',
      totalLabel:        true      // show sum in centre hole
    }
  }
});

Centre Total Label Styling

The centre total is a <div> with class jqplot-data-label but without the series class — target it with a :not() selector: 

.jqplot-donut-series.jqplot-data-label {
  font-weight: bold;
  color: #fff;
  text-shadow: 1px 1px 0 #000, -1px -1px 0 #000,
               1px -1px 0 #000, -1px  1px 0 #000;
}

/* Centre total only */
.jqplot-data-label:not(.jqplot-pie-series):not(.jqplot-donut-series) {
  font-weight: bold;
  font-size:   1.1rem;
  color:       #2c3e50;
}
The centre total label positioning is handled by the renderer — it adds canvas._offsets and subtracts half the element dimensions to centre it correctly. Do not add transform: translate(-50%, -50%) in CSS or the label will be double-offset.

Three Compounding Bugs Fixed in This Fork

The original jqPlot donut renderer had three interacting bugs in drawSlice() that caused invisible rings. All three are fixed in this fork:

BugSymptomFix
Lost this context this.startAngle resolved to window.startAngle (undefined) inside nested doDraw() Capture var self = this before entering doDraw()
Unit mismatch Angle clamp compared radians against startAngle stored in degrees, collapsing all arcs Convert to radians: var saRad = self.startAngle / 180 * Math.PI
Closure mutation Shadow loop called doDraw() multiple times, corrupting shared ang1/ang2 Use local copies: var a1 = ang1, a2 = ang2 inside doDraw()
Fixing Bug 1 without fixing Bug 2 made things worse — the clamp started firing with the correct object but the wrong units. All three required coordinated fixes.
