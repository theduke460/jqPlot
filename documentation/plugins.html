Interaction Plugins

Highlighter, Cursor, Canvas Overlay, Draggable, and Point Labels.

One plugin per mouse-event stream: The highlighter and cursor plugins both bind to mousemove on the event canvas and compete for the same events. Enable only one per chart. Set { show: false } on whichever you are not using.

Highlighter — Hover Tooltips

Shows a floating tooltip when the mouse is near a data point marker. sizeAdjust enlarges the invisible hit zone around each marker. tooltipAxes controls which axis values appear in the tooltip. 

<script src="dist/plugins/jqplot.highlighter.min.js"></script>

$.jqplot('chart', [data], {
  highlighter: {
    show:               true,
    sizeAdjust:         7.5,
    tooltipAxes:        'both',     // 'x' | 'y' | 'both'
    bringSeriesToFront: true,
    formatString:       '<strong>x=%s  y=%s</strong>'
  },
  cursor: { show: false }
});

formatString Tokens

The format string uses %s placeholders. With tooltipAxes: 'both' the first %s is the x value and the second is the y value. With tooltipAxes: 'y' only one %s is needed. 

// Show only y value:
formatString: '<strong>%s M users</strong>'

// Show both with labels:
formatString: 'Sample %s → %s ms'

// Date axis — x value will be a formatted date string:
formatString: '<strong>%s &nbsp; $%s M</strong>'

Cursor — Crosshair with Coordinate Readout

Draws a crosshair at the mouse position with a live coordinate readout. useAxesFormatters: true delegates number formatting to each axis's own tick formatter — this is the correct way to get properly-scaled values in the tooltip. 

<script src="dist/plugins/jqplot.cursor.min.js"></script>

$.jqplot('chart', [data], {
  cursor: {
    show:                    true,
    showTooltip:             true,
    followMouse:             true,
    showVerticalLine:        true,
    showHorizontalLine:      true,
    // Delegate formatting to each axis's tick formatter — avoids unit mismatch:
    useAxesFormatters:       true,
    showTooltipUnitPosition: true,
    showTooltipGridPosition: false,
    style:                   'crosshair'
  },
  highlighter: { show: false }
});
Tooltip value mismatch: Using tooltipFormatString with custom printf tokens (%.1f%%) applies the format to raw internal coordinate values, not to axis-scaled values. Always use useAxesFormatters: true to get the same values shown on the axis ticks.

Y-Axis Autoscaling with Cursor

When using the cursor plugin, avoid hardcoding min/max on the y-axis. If data can exceed those bounds the series clips silently: 

yaxis: {
  // No min/max — autoscale
  padMin: 1.03,
  padMax: 1.03,
  tickOptions: { formatString: '%d%%' }
}

Canvas Overlay — Arbitrary Lines and Regions

Draws reference lines and regions on top of any chart after it renders. All coordinates are in data units (same scale as the axes), not pixels. The plugin converts them internally via series_u2p(). 

<script src="dist/plugins/jqplot.canvasOverlay.min.js"></script>

$.jqplot('chart', [data], {
  canvasOverlay: {
    show: true,
    objects: [
      // Solid horizontal line at y=145
      { horizontalLine: {
          name:     'buy-zone',
          y:         145,
          lineWidth: 2,
          color:    '#27ae60',
          shadow:    false
      }},
      // Solid horizontal line at y=180
      { horizontalLine: {
          name:     'sell-zone',
          y:         180,
          lineWidth: 2,
          color:    '#e74c3c',
          shadow:    false
      }},
      // Dashed horizontal line
      { dashedHorizontalLine: {
          name:        'target',
          y:            195,
          lineWidth:    2,
          color:       '#e67e22',
          dashPattern: [8, 5],
          shadow:       false
      }}
    ]
  }
});

Available Object Types

TypeRequired options
horizontalLiney — data-unit y value
dashedHorizontalLiney, dashPattern: [dashLen, gapLen]
verticalLinex — data-unit x value
dashedVerticalLinex, dashPattern
linestart: [x1,y1], stop: [x2,y2]

Draggable — Interactive Data Editing

Makes individual series markers draggable. Opt in per series with isDraggable: true. The jqplotDataChange event fires continuously during a drag with the updated [x, y] value. 

<script src="dist/plugins/jqplot.draggable.min.js"></script>

$.jqplot('chart', [sensorData], {
  series: [{
    isDraggable: true,          // opt-in — note: two g's
    markerOptions: { size: 10, style: 'filledCircle' }
  }],
  draggable: {
    show:        true,
    constrainTo: 'y'            // 'x' | 'y' | 'none'
  }
});

// Handle drag events:
$('#chart').on('jqplotDataChange', function(evt, seriesIdx, pointIdx, data) {
  console.log('Series', seriesIdx, 'Point', pointIdx, '→', data);

  // Optional: debounced Ajax save
  clearTimeout(window._dragTimer);
  window._dragTimer = setTimeout(function() {
    $.ajax({
      url:  '/api/update',
      method: 'POST',
      data: JSON.stringify({ series: seriesIdx, point: pointIdx,
                             x: data[0], y: data[1] })
    });
  }, 300);
});
The property name is isDraggable (two g's). The plugin file is named jqplot.draggable.js. An older misspelled variant jqplot.dragable.js also exists in the dist; loading it and using isDragable (one g) will not work with the canonical plugin.

Use Cases for Draggable Data

The plugin updates the series .data array in-place. Useful for:

  • Forecast adjustment — analysts drag projected values to apply manual overrides
  • Threshold setting — operations teams drag SLA limits to new values
  • Capacity planning — managers drag staffing levels; server recalculates costs
  • Sensor calibration — technicians drag reference points to correct instrument drift

Point Labels

Annotates each data point marker with a formatted value label. Works with line, bar, and scatter charts. 

<script src="dist/plugins/jqplot.pointLabels.min.js"></script>

seriesDefaults: {
  pointLabels: {
    show:         true,
    location:     'n',          // 'n' | 's' | 'e' | 'w' | 'ne' | 'nw' etc.
    ypadding:     5,            // gap between marker and label
    formatString: '%.1fM'       // sprintf format for the value
  }
}

HTML Labels with Custom Styling

Set escapeHTML: false and provide an explicit labels array of HTML strings to apply custom styles — useful for white text with black outlines inside colored bars: 

pointLabels: {
  show:          true,
  location:      'n',
  escapeHTML:    false,
  edgeTolerance: -100,          // prevent suppression on short bars
  labels: data.map(function(v) {
    return '<span style="font-weight:bold;color:#fff;' +
      'text-shadow:1px 1px 0 #000,-1px -1px 0 #000">' + v + '</span>';
  })
}
