Date Enhancement Plugin

jqplot.dateEnhancement.js — shaded date/value ranges and annotated point events on any jqPlot chart.

This plugin works on any jqPlot chart — not just date-axis charts. Pass date strings for DateAxisRenderer axes; pass numeric x-values for linear axes.

Loading

<!-- Load after dateAxisRenderer if using date axes -->
<script src="dist/plugins/jqplot.dateAxisRenderer.min.js"></script>
<script src="dist/plugins/jqplot.dateEnhancement.min.js"></script>

The plugin self-registers via $.jqplot.preParseOptionsHooks and $.jqplot.postDrawHooks. No explicit activation call is needed. It activates automatically when dateRanges or dateEvents keys are present in the chart options object.

Icon Convention

The icon shape carries semantic meaning and is enforced programmatically:

Circle — point-in-time event (dateEvents) Diamond — collapsed narrow range (auto-converted from dateRanges)

Callers cannot override the shape. This ensures the visual language is consistent across all charts that use the plugin.

dateRanges — Shaded Regions

Each entry in the dateRanges array draws a semi-transparent shaded rectangle spanning the x range and the full grid height.

Wide ranges (≥ 100px)

Drawn as a shaded rectangle with an inline label at the top-left corner.

Narrow ranges (< 100px)

Automatically converted to a diamond event marker positioned at the range midpoint. The label is built programmatically as:
"<name>\n<startDate> – <endDate>"
This eliminates cramped or overflowing labels without any data changes. The threshold is configurable via dateNarrowThreshold.

Example

$.jqplot('chart', [data], {
  axes: {
    xaxis: { renderer: $.jqplot.DateAxisRenderer, /* ... */ }
  },

  dateRanges: [
    // Wide range — will show shaded rectangle + inline label
    { start:      '1939-09-01',
      end:        '1945-09-02',
      color:      '#27ae60',
      opacity:     0.18,
      label:      'WW2',
      labelColor: '#1e8449',
      labelSize:   11
    },
    // Narrow range — will auto-collapse to a diamond event
    { start:      '1960-04-01',
      end:        '1961-02-01',
      color:      '#2980b9',
      opacity:     0.20,
      label:      'Recession',
      labelColor: '#1a5276',
      labelSize:   9
    }
  ]
});

dateEvents — Point Annotations

Each entry draws a circle marker at the given x-axis value. Y positions are spread programmatically across a configurable band using stratified placement — events are distributed into equal slots across the band, with seeded random jitter within each slot for a natural look without collision.

Positions are stable within a session (same seed = same position on redraws) but vary between page loads because a page-load timestamp is mixed into every seed at plugin load time.

Example

dateEvents: [
  { date:      '1969-07-20',
    label:     'Apollo 11 Moon Landing\nJul 20, 1969',
    color:     '#2c3e50',
    size:       9,
    labelSize:  10
  },
  { date:      '1986-01-28',
    label:     'Challenger Disaster\nJan 28, 1986',
    color:     '#e74c3c',
    size:       7,
    labelSize:  10
  }
]

Two-Line Labels

Use \n in the label string to split it across two lines. The plugin renders labels with white-space: pre-line so the line break is honoured. Both lines share the same color and background. 

// Single-line:
label: 'Apollo 11 Moon Landing'

// Two-line (title + date):
label: 'Apollo 11 Moon Landing\nJul 20, 1969'
In JavaScript source, \n must be written as two characters (backslash + n) inside a string literal. A literal newline character inside a single-quoted string is a syntax error.

Numeric Axis Usage

Use the x key instead of date for numeric axes: 

// On a chart with a numeric x-axis (samples 1–10):
dateEvents: [
  { x: 4, label: 'Cache Miss Spike', color: '#e74c3c', size: 7 },
  { x: 8, label: 'CDN Failover',     color: '#8e44ad', size: 7 }
]

Hover Tooltips

All markers — both point-event circles and collapsed-range diamonds — show a floating tooltip on mouse hover. The tooltip displays the full label text, including both lines of two-line labels. It is color-coded with a left border matching the marker color.

The tooltip is implemented as a single shared <div> managed by the plugin. A mousemove handler on the jqPlot event canvas tests the cursor's proximity to each recorded marker position (within markerSize + 4 px). The cursor changes to a pointer when over a hit zone.

No configuration is required — hover tooltips are always active when the plugin is loaded.

Y Scatter — How It Works

Event Y positions use stratified placement: the scatter band is divided into N equal slots (one per event, including collapsed-range events). Each event is assigned to its slot by index % N, then a seeded pseudo-random value provides jitter within the slot. This guarantees visual spread regardless of how many events there are or what their labels say. 

function stratifiedY(index, total, bandTop, bandBottom, seed) {
  var slotH    = (bandBottom - bandTop) / total;
  var slotTop  = bandTop + (index % total) * slotH;
  var jitter   = seededRandom(seed) * slotH * 0.7;
  return slotTop + slotH * 0.15 + jitter;
}

Seeds are derived from labelSeed(label) + eventIndex + PAGE_LOAD_SEED. The PAGE_LOAD_SEED is Math.floor(Date.now() / 1000) & 0xFFFF, evaluated once when the plugin script loads. This means:

  • Positions are stable within a session — redraws don't move markers
  • Positions vary between page loads — each load gets a different scatter
  • Callers can override with an explicit seed for fully reproducible placement

Implementation Notes

How the plugin hooks in

// preParseOptionsHooks: adjust gridPadding.top before layout is calculated.
// Signature: fn.call(plotObj, options) — one argument only.
$.jqplot.preParseOptionsHooks.push($.jqplot.DateEnhancement.preParseOptions);

// postDrawHooks: draw overlays after all canvas drawing is complete.
// Signature: fn.call(plotObj) — no arguments, use 'this'.
$.jqplot.postDrawHooks.push($.jqplot.DateEnhancement.postPlotDraw);
Hook signature pitfall: preParseOptionsHooks passes only (options) to the handler, not (target, data, options). Declaring extra parameters causes options to receive the wrong value, which crashes initialization for all charts on the page.

Coordinate conversion

// Inside postPlotDraw — convert x-axis value to canvas pixel:
function xPx(val) {
  return plot._gridPadding.left + plot.axes.xaxis.series_u2p(resolveX(val, xaxis));
}

// resolveX: DateAxisRenderer axes need the value as a timestamp.
// By postDraw time, axis.renderer is an INSTANCE — use instanceof:
function resolveX(val, xaxis) {
  if (xaxis.renderer instanceof $.jqplot.DateAxisRenderer) {
    return new $.jsDate(val).getTime();
  }
  return Number(val);
}

Label width measurement

To decide whether a range label fits inside its shaded region, the plugin inserts a hidden clone of the label into the DOM, measures its rendered pixel width with outerWidth(), then removes it. This is the only reliable method — character-count estimates vary with font, size, and browser. 

function measureLabel(text, fontSize, target) {
  var $probe = $('<div></div>').text(text).css({
    position: 'absolute', visibility: 'hidden',
    whiteSpace: 'nowrap', fontSize: fontSize + 'px',
    fontWeight: 'bold', padding: '2px 5px'
  });
  target.append($probe);
  var w = $probe.outerWidth(true);
  $probe.remove();
  return w;
}
