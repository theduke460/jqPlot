Axis Types

Linear, logarithmic, date/time, and category axis renderers.

Linear Axis (Default)

The default axis renderer. Evenly-spaced numeric ticks. Configure tick format, min/max, and label via the axis options object. 

axes: {
  yaxis: {
    label:         'Revenue ($M)',
    labelRenderer: $.jqplot.CanvasAxisLabelRenderer,
    min:   0,
    padMax: 1.05,               // 5% breathing room above max data value
    tickOptions: {
      formatString: '$%.1fM'    // sprintf format
    }
  }
}
CanvasAxisLabelRenderer and CanvasAxisTickRenderer are required for angled tick labels. Always load canvasTextRenderer first: 
<script src="dist/plugins/jqplot.canvasTextRenderer.min.js"></script>
<script src="dist/plugins/jqplot.canvasAxisLabelRenderer.min.js"></script>
<script src="dist/plugins/jqplot.canvasAxisTickRenderer.min.js"></script>

Logarithmic Axis

Use LogAxisRenderer when data spans multiple orders of magnitude. Each major tick represents a power of the base (default 10). Both axes can be log-scaled independently for a log-log plot. 

<script src="dist/plugins/jqplot.logAxisRenderer.min.js"></script>

axes: {
  yaxis: {
    renderer:         $.jqplot.LogAxisRenderer,
    label:            'Colony Count (log scale)',
    labelRenderer:    $.jqplot.CanvasAxisLabelRenderer,
    // base: 10 is default — each gridline = 10× increase
    tickDistribution: 'even',   // ticks at 10^n values
    tickOptions: { formatString: '%\'d' }
  }
}
Log of zero is undefined. The axis min must always be greater than zero. Data points at or below zero will produce errors or invisible series.

Log-Log Plot

Apply LogAxisRenderer to both axes for a log-log chart. The inverse-square law (P ∝ 1/d²) produces a straight line on a log-log scale:

axes: {
  xaxis: {
    renderer:         $.jqplot.LogAxisRenderer,
    tickDistribution: 'even',
    tickOptions:      { formatString: '%dm' }
  },
  yaxis: {
    renderer:         $.jqplot.LogAxisRenderer,
    tickDistribution: 'even',
    tickOptions:      { formatString: '%.4f' }
  }
}

Date Axis

DateAxisRenderer parses date strings and converts them to millisecond timestamps internally. Data points are ['YYYY-MM-DD', value] pairs. Tick labels are formatted with strftime codes. 

<script src="dist/plugins/jqplot.dateAxisRenderer.min.js"></script>

var data = [
  ['2020-01-01', 18.9],
  ['2021-01-01', 24.7],
  ['2022-01-01', 28.2],
  ['2023-01-01', 31.5]
];

axes: {
  xaxis: {
    renderer:     $.jqplot.DateAxisRenderer,
    tickInterval: '1 year',         // '1 year' | '6 months' | '1 day' etc.
    tickOptions:  { formatString: '%Y', angle: -30 },
    label:        'Year',
    labelRenderer: $.jqplot.CanvasAxisLabelRenderer
  }
}

Date Conversion Inside Plugins

After DateAxisRenderer.init() runs, all x-values in the series data are converted to millisecond timestamps via new $.jsDate(val).getTime(). When writing plugins that need to convert date strings to pixel positions: 

// Inside a postDrawHooks handler:
var ts  = new $.jsDate('2024-06-01').getTime();
var xPx = plot._gridPadding.left + plot.axes.xaxis.series_u2p(ts);

// Detect DateAxisRenderer at hook time (renderer is an INSTANCE, not a class):
var isDateAxis = (xaxis.renderer instanceof $.jqplot.DateAxisRenderer);
By the time postDrawHooks fire, axis.renderer is an instance of the renderer class, not the class itself. Use instanceof — not === — for type detection.

Category Axis

Assigns one evenly-spaced slot per data point with no numeric scale implied. String tick labels come from the ticks array. The primary use case is bar charts where x-axis categories are labels, not numbers. 

<script src="dist/plugins/jqplot.categoryAxisRenderer.min.js"></script>

axes: {
  xaxis: {
    renderer:    $.jqplot.CategoryAxisRenderer,
    ticks:       ['Engineering','Marketing','Operations','HR','Sales'],
    tickOptions: { angle: -25 }   // rotate labels to avoid overlap
  }
}

Category Axis on Horizontal Bar Charts

For horizontal bars, the y-axis gets CategoryAxisRenderer. Use blank guard ticks at positions 1 and N to prevent bars touching axis edges. Bar data y-indices must match the named tick positions exactly. 

// Ticks: blank at 1, named at 2–6, blank at 7
// Bar data uses y = 2,3,4,5,6 to land on the named positions
axes: {
  yaxis: {
    renderer: $.jqplot.CategoryAxisRenderer,
    ticks: ['', 'Discovery', 'Design', 'Development', 'Testing', 'Deployment', ''],
    pad: 1.0
  }
}
NaN coordinates: Passing plain string arrays as ticks on a LinearAxisRenderer axis causes tickInterval = NaN and makes all series coordinates NaN. Canvas silently ignores lineTo(NaN, y), so the series disappears while DOM elements (legend, axis labels) still render. Always use CategoryAxisRenderer for string-labelled axes.
